# 05 Schedule Management Plan

## Sprint Model

The project is executed in nine focused sprints. Each sprint should end with a working build before the next sprint begins.

## Sprint Order

1. Audit and Foundation
2. Authentication Upgrade
3. Mobile Navigation
4. Dashboard Mobile Overhaul
5. Transition Timeline Mobile Overhaul
6. Tools Mobile Overhaul
7. Library and Knowledge Base Mobile Overhaul
8. Forms and Upload Warnings
9. QA and Hardening

## Exit Criteria Per Sprint

- Scope completed or deferred intentionally
- `npm run build` completed or blocker documented
- Files changed summarized
- Remaining issues captured
- Next sprint ready to start
