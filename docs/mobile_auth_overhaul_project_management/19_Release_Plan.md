# 19 Release Plan

## Release Name

Mobile Auth Overhaul Release

## Release Objective

Ship a mobile-first MilVector AI experience with email/password authentication and preserved magic-link access.

## Release Strategy

- Incremental sprint implementation
- Build verification after every sprint
- Small Git commits
- No broad refactor without sprint justification

## Release Readiness Criteria

- Production build passes
- Auth flows verified
- Magic link remains available
- Mobile navigation works at 360px
- Dashboard and timeline are usable on phones
- AI tools are usable on phones
- Upload warning appears on every upload surface
- No major Supabase/RLS regression
