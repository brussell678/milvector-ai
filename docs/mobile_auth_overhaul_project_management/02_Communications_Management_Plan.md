# 02 Communications Management Plan

## Purpose

Set communication expectations for sprint execution.

## Communication Principles

- Use concise sprint updates.
- Keep technical summaries grounded in changed files and user-visible behavior.
- Separate completed work from remaining issues.
- Call out manual Supabase, Vercel, or Cloudflare checks clearly.

## Standard Sprint Closeout Format

- Files changed
- Summary of behavior changed
- Verification performed
- Acceptance criteria status
- Remaining issues
- Recommended next sprint or follow-up

## Escalation Triggers

- Build failure that cannot be resolved inside the sprint scope
- Supabase schema mismatch
- Auth redirect behavior that requires dashboard/provider configuration
- Mobile overflow caused by shared layout assumptions
- Any user data or RLS risk
